This project inherits the MD operating discipline from `../md/CLAUDE.md` and the default

## The project, in one paragraph
Predict **B-cell epitope residues from antigen-only information**. Residue-level labels are
alanine-scan binding-importance values, Sᵢ ≈ ΔΔG_bindᵢ (experimental ground truth, curated
in `benchmark/`). We simulate the **apo antigen** and ask whether dynamics/solvation/energetic
features (RMSF, FRESEAN mode participation, 3D-2PT hydration, intra-protein energetics) predict
binding importance — and specifically whether they add signal **beyond static surface exposure
(SASA/depth)**. First system: hen egg-white lysozyme (HEL, 1AKI), which carries three antibody
alanine scans (HyHEL-10, HyHEL-63, D1.3). PI: John Altin. Author's background: MD / FRESEAN /
non-equilibrium energy transfer (`../_literature/my_manuscripts/`).

## How to work here (the contract)
- **Reproducibility over speed — always.** Never trade reproducibility for a faster result.
- **The LaTeX log is the living record.** `manuscript/` is read top-to-bottom: each section is
  one iteration — what we tried, what we found, what it changed. When a result changes, update
  the section text, the figure, and the command that produced it the *same* session. Every
  figure is regenerated by a named script recorded in `manuscript/figures/FIGURES.md`; every
  number traces to a file or command. Flag unconfirmed values `[verify]`.
- **No silent methodological choices.** See `../md/docs/METHODS_DEFAULTS.md`. This project is
  dynamics/solvation-sensitive, so defaults (esp. TIP3P, 2 fs, LINCS, single thermostat) get
  real scrutiny per run type — flexibility runs differ from FRESEAN/3D-2PT runs.
- **Freeze inputs.** Each simulation owns frozen `.mdp`/scripts under `md/<run>/configs/`. Don't
  run against a moving shared file.
- **Record every run.** Command, GROMACS version, FF+water, structure provenance, seeds, salt,
  box, Slurm job id — into the manuscript run-log section. Unrecorded ⇒ not reproducible.
- **Verify against data, not memory.** Residue ids, ΔΔG values, chains, columns — read them from
  the actual files and show the command. Never invent them. (Benchmark labels: `benchmark/`.)
- **Leakage discipline (ML).** Split leave-one-complex-out / leave-one-fold-out, never random
  residue splits. Effective N ≈ #antigens, not #residues. Report honestly.
- **One step at a time; plan before large/destructive actions.** This repo IS under git, but
  `/tgen_labs` and Gemini data are not — never delete or `rsync --delete` toward them.

## Where things live
- `manuscript/` — living LaTeX log (`main.tex` + `sections/` + `figures/` + `FIGURES.md` + `refs.bib`).
- `benchmark/` — curated alanine-scan labels (`antigen_alanine_scanning_benchmark_v2.xlsx`) + audit.
- `structures/` — input + cleaned antigen structures (e.g. `1AKI.pdb`).
- `md/<run>/` — simulation work: `configs/` (frozen mdp) + sbatch; outputs gitignored (live on Gemini).
- `analysis/` — feature extraction + correlation/ML code.
- `docs/` — `METHODS.md` (project-frozen stack), `ENVIRONMENT_gemini.md` (cluster).
